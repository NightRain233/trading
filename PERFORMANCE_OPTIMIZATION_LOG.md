这是一次从 **3.5秒** 到 **47毫秒**（性能提升约 **75倍**）的深度调优记录。

## 🛠 五个进化阶段

### 1. 深度切片与病因诊断 (Profiling)
*   **诊断**：通过引入上下文计时器 `timer` 和细分日志发现，瓶颈不在网络下载，而是在即使有日线缓存时，程序仍在重复计算“周线指标”。
*   **痛点**：单线程处理 21 只股票的数学指标计算（EMA, MACD, RSI）产生了巨大的 CPU 开销。

### 2. 多任务并行化 (Parallelism)
*   **尝试**：引入 `ThreadPoolExecutor` 将单线程改为多线程。
*   **反思**：由于 Python GIL 锁的存在，对于 CPU 密集型计算提升不明显，这引导我们转向“减少计算”而非“加速计算”。

### 3. 二级持久化缓存 (Multi-layer Disk Cache)
*   **方案**：新增 `*_weekly.parquet` 文件，将计算繁重的周线指标直接持久化。
*   **成果**：总耗时从 **2.8s 降至 400ms**，实现了“不重启即极速”。

### 4. 内存一级缓存与后台管家 (L1 Cache & Background Warming)
*   **内存缓存 (In-Memory)**：引入全局内存字典，读取速度从硬盘级的毫秒降至内存级的纳秒。
*   **后台管家 (Background Refresh)**：实现了 20 分钟一次的后台自动刷新，确保用户访问时缓存永远是“热”的。
*   **旧数据优先 (Stale-While-Revalidate)**：即使缓存刚过期，也先秒回旧数据，后台同步更新，消灭了任何感知的卡顿。

### 5. 零计算直出 (Zero-Cost Summary)
*   **优化**：将最终的 JSON 摘要字典直接预存在内存中。
*   **成果**：接口逻辑时间降至 **8ms** 左右，达到了工业级的极致响应。
